# Knowledge Sheets Template

เอกสารนี้ใช้เป็นแม่แบบสำหรับจัด Google Sheet ให้ตอบตรงประเด็นมากขึ้น และลดอาการที่ AI ดึงหลายแถวมารวมกันจนตอบกว้างเกินไป

## โครงสร้างที่แนะนำ

ให้ทุกแท็บใช้หัวคอลัมน์แบบเดียวกัน:

```text
question | answer | keywords | intent | active
```

## ความหมายของแต่ละคอลัมน์

- `question`
  คำถามที่ user มักถามจริง 1 แถวต่อ 1 คำถาม
- `answer`
  คำตอบหลักแบบสั้น กระชับ ตรงประเด็น ไม่ควรยาวเกิน 2-3 ประโยค
- `keywords`
  คำใกล้เคียงหรือวลีที่ user ชอบพิมพ์ เช่นคำไทย คำอังกฤษ คำเรียกอีกแบบ
- `intent`
  เจตนาของคำถาม เช่น `definition`, `pricing`, `required_info`
- `active`
  ใช้ `yes` หรือ `no`

หมายเหตุ:
- ถ้ายังไม่พร้อมใช้ `intent` ตอนนี้ ระบบยังอ่านได้ แต่แนะนำให้เพิ่มไว้เลย
- ถ้าแถวนั้นไม่ต้องการใช้งาน ให้ใส่ `active = no`
- ใช้เฉพาะ intent ที่ระบบรองรับจริงตาม [GOOGLE_SHEETS_INTENT_GUIDE.md](/C:/Users/Sorravit_L/Frieght/docs/GOOGLE_SHEETS_INTENT_GUIDE.md)

## หลักการเขียนที่ช่วยให้ตอบตรงขึ้น

- 1 แถว = 1 คำถาม = 1 คำตอบ
- อย่ารวมหลายเจตนาไว้ในแถวเดียว
- `keywords` ควรเป็นวลี ไม่ใช่แค่คำเดี่ยว
- ถ้าเป็นเรื่องเดียวกันแต่ user ถามต่างแบบ ให้เพิ่มหลายแถว
- ใช้คำที่ user พูดจริง ไม่ต้องเป็นภาษาทางการเกินไป

ตัวอย่าง:

ไม่ควร:

```text
บริการ Solar ผ่าน Hub คืออะไร เหมาะกับงานแบบไหน ราคาเท่าไหร่ ต้องใช้อะไรบ้าง
```

ควรแยกเป็น:

```text
บริการ Solar ผ่าน Hub คืออะไร
งานแบบไหนเหมาะกับ Solar ผ่าน Hub
Solar ผ่าน Hub คิดราคายังไง
ถ้าจะใช้ Solar ผ่าน Hub ต้องเตรียมข้อมูลอะไรบ้าง
```

## วิธีกรอกให้ search แม่นขึ้น

1. ใช้คำถามที่ user พูดจริง
2. ใส่ `keywords` เป็นวลี เช่น `ขอราคาใช้อะไรบ้าง` ดีกว่าใส่แค่ `ราคา`
3. ถ้าคำถามเดียวกันมีหลายสำนวน ให้แยกหลายแถวได้
4. ถ้าคำตอบคนละเรื่อง อย่ารวมในแถวเดียว
5. ถ้าข้อมูลเลิกใช้แล้ว ให้ใส่ `active = no`
