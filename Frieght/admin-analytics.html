weekly summary

สรุปภาพรวมช่วงวันที่เลือกแบบไว ๆ ให้หัวหน้าหรือทีมดูว่าปริมาณงาน งานค้าง งานที่ปิดแล้ว และ feedback ไปทางไหนบ้าง

top unresolved reasons

ดูเหตุผลหลักที่ทำให้คิวยังเปิดอยู่บ่อยสุด จะได้รู้ว่าควรแก้ flow ไหนหรือเติม knowledge หมวดไหนก่อน

daily review workflow

ใช้เป็นคิวเช็กทุกวันว่า วันนี้ทีมควรไล่รีวิวอะไร ปิดเคสอะไร และ approve draft ไหนลงชีตก่อน

-

feedback ยังไม่ตรงวันนี้
-

approve ลงชีตวันนี้
-

priority intents วันนี้

approve ล่าสุด

owner dashboard

ดูภาพรวมงานรีวิวของแต่ละคนว่าใครคิวเปิดเยอะ ใครปิดเคสแล้วเยอะ และใคร approve ลงชีตไปเท่าไร จะได้กระจายงานกันง่ายขึ้น

agent productivity

สรุปภาพรวมต่อคนว่าใครมีงาน active เท่าไร วันนี้ขยับเคสไปแล้วกี่ครั้ง และฝั่ง review/handoff ไปทางไหนบ้าง

handoff queue

รวมเคสที่ user ฝากให้ทีมติดต่อกลับจากหน้าแชต จะได้ไม่หลุดตอนบอทคุยไม่พอหรือ user อยากคุยกับคนจริงต่อ

-

knowledge automation

ดูสถานะ sync ความรู้ล่าสุด แล้วกด sync จากหน้าแอดมินได้เลย เวลาทีมเติมชีตแล้วอยากให้ขึ้น bot เร็ว ๆ

SLA dashboard

ดูอายุเคสที่ยังเปิดอยู่ ว่ามีงานค้างเกิน 1 วันหรือ 3 วันเท่าไร และมีเคสไหนที่ควรรีบหยิบขึ้นมาปิดก่อน

เปิดไม่ถึง 1 วัน
-
คิวใหม่ที่ยังตามได้ทัน ไม่ควรปล่อยให้เลย SLA
ค้าง 1-3 วัน
-
เริ่มควรหยิบมาดูแล้ว ถ้ามี owner ให้ follow up ต่อได้เลย
ค้างเกิน 3 วัน
-
กลุ่มนี้ควรรีบปิดก่อน เพราะมีโอกาสหลุดงานจริงสูงสุด

activity timeline

ไล่เหตุการณ์ล่าสุดของคิวรีวิว ว่าเพิ่งมี feedback, เปลี่ยนสถานะ review, หรือ approve ลงชีตเมื่อไหร่บ้าง

knowledge health

ดูว่า intent ไหนมี log เยอะ แต่ knowledge ยังน้อย หรือโดน feedback ว่ายังไม่ตรงบ่อย เพื่อใช้จัดลำดับปรับชีต

Intent ที่ถูกถามบ่อย

ดูว่า user สนใจหมวดไหนจริง แล้วค่อยเอาไปเติมชีตให้ตรงงาน

Source / Lane ที่ระบบใช้ตอบ

เอาไว้ดูว่า backend ตอบจาก tracking, knowledge, หรือ model มากแค่ไหน

คำถามที่ถามซ้ำบ่อย

ถ้ากลุ่มนี้เริ่มเยอะ แปลว่าควรทำ intent หรือ answer ใน Google Sheet ให้เฉพาะขึ้นอีก

เคสที่น่าหยิบไปรีวิวต่อ

ดูตัวอย่างข้อความที่มีโอกาสพลาด เพื่อเอาไปเพิ่ม keywords, intent, หรือ answer ในชีตต่อ

top failed questions

ดูเฉพาะคำถามที่เคยเข้า fallback, not found, หรือ error เพื่อรู้ว่าจุดไหนควรรีบเติมชีตก่อน

review queue

คิวนี้โชว์เฉพาะเคสที่ยังเปิดอยู่ ถ้าแก้ชีตหรือ keywords แล้ว กดติ๊กว่าแก้แล้วได้เลย

หยิบไปเติม Google Sheet ต่อได้เลย

รายการนี้ช่วยสรุปคำถามที่ควรเอาไปเติมชีตต่อให้เร็วขึ้น โดยมี topic / intent ที่แนะนำ และมีแถวแบบ copy-ready ให้ใช้ต่อได้ทันที

log ล่าสุด
